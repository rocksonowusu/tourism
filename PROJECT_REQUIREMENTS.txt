================================================================================
  MISSION CONTEXT
================================================================================

  "At 1957 The Ghana Experience LBG, our mission is to deliver authentic and
   unforgettable travel experiences that showcase the rich culture, heritage,
   and natural beauty of Ghana."

  KEY BUSINESS RULES:
  • The site is INFORMATIONAL — no payment processing, no online booking.
  • All customer actions (trip planning, apartment requests, event requests,
    car rental inquiries) result in:
      1. An EMAIL sent to the company's email address, AND
      2. A record/notification appearing in the Admin Portal, AND/OR
      3. Opening WhatsApp with a PRE-FILLED message based on user selections.
  • Media (images/videos) is central to every feature — everything has
    corresponding media files uploaded via the admin portal.

================================================================================
  PHASE 2 — TOUR TRIP PLANNING & REQUEST SYSTEM
================================================================================

  REQUIREMENT: Customers should be able to plan a trip by selecting from
  available tours, entering traveller details by age group, and submitting a
  request. This sends an email to the company AND creates a record in the
  admin portal. Option to open WhatsApp with a pre-filled message.

  Tasks:
  ────────────────────────────────────────────────────────────────────────────

  BACKEND:
  2.1  Tour/Package Model
       • Create a Tour model (or repurpose/extend Event) with fields:
         – title, slug, description, location, duration, price_per_adult,
           price_per_child, price_per_infant, highlights, inclusions,
           exclusions, itinerary (JSON), is_active, is_featured,
           max_group_size, media (related).
       • Create TourMedia model (images/videos per tour, Cloudinary).

  2.2  Trip Request Model
       • Create a TripRequest model with fields:
         – tour (FK), customer_name, customer_email, customer_phone,
           number_of_adults, number_of_children (ages 3-12),
           number_of_infants (under 3), preferred_date, special_requests,
           status (new/contacted/confirmed/cancelled), created_at.
       • Admin can view, filter, update status of requests.

  2.3  Trip Request API
       • POST /api/trip-requests/ — PUBLIC (no auth required) to submit.
       • GET /api/trip-requests/ — ADMIN only, with filtering & pagination.
       • PATCH /api/trip-requests/{id}/ — ADMIN only, update status.

  2.4  Email Notification
       • On trip request submission, send a formatted email to the company
         email with all trip details (tour name, dates, group breakdown,
         contact info, special requests).
       • Send a confirmation email to the customer.

  FRONTEND:
  2.5  Public Site — Trip Planning Page/Flow
       • New page/section: "Plan Your Trip"
       • Display available tours with images, price, duration.
       • Tour detail view with full info + "Request This Tour" form.
       • Form fields: name, email, phone, preferred date, number of adults,
         children, infants, special requests/notes.
       • On submit → call API → show success message.
       • "Chat on WhatsApp" button that opens WhatsApp with a pre-filled
         message summarising the selected tour and group details.

  2.6  Admin Portal — Trip Requests Management
       • New admin page: "Trip Requests"
       • Table listing all requests with filters (status, date, tour).
       • Click to view full request details.
       • Update status (new → contacted → confirmed → cancelled).
       • Notification/badge showing new unread requests.

  2.7  Tours Management in Admin
       • CRUD for Tours (similar to existing Events CRUD).
       • Upload media for each tour.

  Deliverables:
       • Tour & TripRequest models + migrations.
       • API endpoints for tours (CRUD) and trip requests.
       • Email sending on request submission.
       • Public trip planning UI with WhatsApp integration.
       • Admin trip requests management page.

  Tasks:
  ────────────────────────────────────────────────────────────────────────────

  BACKEND:

================================================================================

  REQUIREMENT: Social media links should be on the site — clicking them opens
  the respective platform. WhatsApp integration throughout for customer
  communication with pre-filled messages.

  Tasks:
  ────────────────────────────────────────────────────────────────────────────

  BACKEND:
  8.1  Company Settings / Social Links
       • Create a SiteSettings or CompanyInfo model (singleton):
         – company_name, phone, email, whatsapp_number,
           facebook_url, instagram_url, twitter_url, tiktok_url,
           youtube_url, linkedin_url, address, about_text.
       • API endpoint to GET company settings (public, read-only).
       • Admin can update these from the admin portal.

  FRONTEND:
  8.2  Social Media Links
       • Add social media icons to the Footer (and optionally Header).
       • Each icon links to the respective platform URL (opens in new tab).
       • Icons: Facebook, Instagram, Twitter/X, TikTok, YouTube, LinkedIn
         (based on what the company uses).

  8.3  WhatsApp Integration (Global)
       • Floating WhatsApp button on all public pages (bottom-right corner).
       • On each service page (tours, apartments, car rentals, events),
         the WhatsApp button generates a pre-filled message like:
           "Hi, I'm interested in [Tour Name]. I'd like to book for
            [X adults, Y children] on [preferred date]. Please let me know
            the availability. Thank you!"
       • WhatsApp link format: https://wa.me/{number}?text={encoded_message}

  8.4  Admin Portal — Settings Page
       • New admin page: "Settings" / "Company Info"
       • Form to update company contact info, social links, WhatsApp number.

  Deliverables:
       • SiteSettings model + API.
       • Social media icons in footer/header.
       • WhatsApp floating button with context-aware pre-filled messages.
       • Admin settings page.

  CROSS-CUTTING CONCERNS (Apply to ALL Phases)
================================================================================

  • RESPONSIVE DESIGN: Every page must work on mobile (≤480px),
    tablet (≤768px), and desktop.
  • MEDIA HANDLING: All new models with media follow the same pattern —
    Cloudinary storage, admin upload via drag-and-drop.
  • EMAIL SYSTEM: Set up Django email backend (SMTP) for all request
    notifications. Use HTML email templates.
  • WHATSAPP MESSAGES: Every request form should have a WhatsApp
    alternative button with a pre-filled message.
  • ADMIN NOTIFICATIONS: All incoming customer requests should show
    a count/badge in the admin sidebar.
  • SEO: All new pages should have proper meta tags, slugs, and
    semantic HTML.
  • NAVIGATION: Update Header/Navbar on public site with links to all
    new sections (Tours, Accommodations, Car Rentals, Community, Reviews).
  • ADMIN SIDEBAR: Update admin sidebar with links to all new management
    pages (Tours, Apartments, Vehicles, Event Requests, Trip Requests,
    Accommodation Requests, Car Rental Requests, Reviews, Settings).
